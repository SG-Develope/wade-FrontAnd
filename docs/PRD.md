## 1. 프로젝트 개요

### 1.1 서비스 목적

낙동강 주변에서 캠핑·낚시·자전거·산책 등 여가를 즐기는 사람들이 실시간 수위 변화를 직접 확인하고, 맑은 날에도 발생할 수 있는 갑작스러운 수위 상승에 스스로 대비할 수 있도록 돕는 수위 모니터링 서비스다.

핵심 가치는 **수위 숫자를 "내가 가려는 장소 기준"으로 번역하는 것**이다. 숫자가 아니라 "지금 가도 되는지"로 안내한다.

### 1.2 서비스 배경

이전 직장에서 공공기관 전용 조기경보 시스템을 운영한 경험이 있다. 해당 시스템은 계측기가 임계값에 도달하면 담당자가 문자·방송을 통해 대피 안내를 송출하는 방식이었다.

이 과정에서 두 가지 문제를 직접 목격했다.

**첫째, 시민은 수동적인 수신자였다.** 기관이 보내주는 알림을 받기 전까지 시민이 스스로 상황을 확인할 수단이 없었다. 기관 알림이 늦거나 누락되면 시민은 위험을 인지하지 못했다.

**둘째, 수위 데이터는 공개되어 있지만 일반인이 읽기 어려웠다.** 낙동강홍수통제소는 실시간 수위 데이터를 공개하고 있지만 숫자와 전문 용어로만 제공되어 캠핑객이나 산책객이 현재 상황이 위험한지 판단하기 어려웠다.

### 1.3 기존 서비스 한계 비교

| 서비스                    | 역할                                              | 한계                                                                          |
| ------------------------- | ------------------------------------------------- | ----------------------------------------------------------------------------- |
| 낙동강홍수통제소 웹사이트 | 실시간 수위 데이터 제공                           | 수치 데이터만, 일반인이 위험 판단 어려움                                      |
| 기상청 날씨 앱            | 날씨·강수 정보                                    | 수위 데이터 없음, 상류 방류로 인한 수위 상승 예측 불가                        |
| 공공기관 조기경보 문자    | 위험 상황 알림                                    | 기관 판단 후 송출, 사용자가 능동적으로 확인 불가                              |
| **본 서비스**             | **수위 + 장소 안전 + CCTV + 날씨 + AI 안내 통합** | **수위를 "내 장소 기준"으로 번역, CCTV로 현장 확인, 레이더로 상류 강수 파악** |

### 1.4 핵심 차별점

**① 기다리는 알림이 아닌, 찾아보는 정보**
기관이 경보를 보내기 전에 시민이 먼저 상황을 판단할 수 있다.

**② 수위 숫자가 아닌, 체감 시각화**

- 평상수위 대비 얼마나 올랐는지 튜브 그래프로 표시
- 제방 단면도로 계획홍수위까지 여유 거리를 직관적으로 표현
- "6.2m" 대신 "평상시보다 +4.4m → 버스 높이만큼 올라옴"으로 전달

**③ 장소 기반 안전 판단**
"양포교 수위 3.7m"가 아니라 "양포교 낚시터 — 지금 주의 필요"로 번역한다.

**④ CCTV + 레이더로 현장 확인 가능 (2주차)**
숫자와 AI 안내만이 아니라 CCTV 영상으로 실제 강 상황을, 기상청 레이더로 상류 강수 현황을 직접 눈으로 확인할 수 있다.

### 2.3 사용 시나리오

**시나리오 1 — 캠핑 전날 안전 확인 (김태민)**

> 김태민은 내일 구미 낙동강 오토캠핑장 예약이 있다. 앱을 열면 여가 지도 탭에서 캠핑장이 "주의" 상태임을 확인한다. 수위 추이 탭에서 양포교 수위가 평상시보다 +2.1m 상승한 것을 직관적으로 확인하고, CCTV 탭에서 현장 영상으로 강 상황을 눈으로 본다. AI 안내에서 "양포교 인근 수위 상승 중, 내일 오전까지 추이를 지켜보세요"라는 메시지를 보고 출발 시점을 조정한다.

**시나리오 2 — 라이딩 전 확인 (박성준)**

> 박성준은 날씨·레이더 탭에서 상류(안동 방향) 강수 레이더를 확인한다. 강수가 없는 것을 확인하고, 여가 지도에서 자전거길이 "안전"인 것을 확인한 뒤 안심하고 출발한다.

**시나리오 3 — 퇴근 후 산책 전 확인 (이수진)**

> 이수진은 실시간 현황에서 호국의다리 관측소가 "정상"인 것을 확인하고 AI 안내에서 "전 구간 정상, 산책하기 좋은 날씨입니다"를 보고 산책을 나선다.

### 3.2 탭 구조

```
┌─ 실시간 현황 탭   지도 + 관측소 수위 카드 + AI 안내           ← 1주차 (route: /dashboard)
├─ 수위 추이 탭     그래프 + 직관 시각화 + 제방 단면도          ← 1주차 (route: /trends)
├─ 여가 지도 탭     4개 장소 + 수위 연동 안전 판단              ← 1주차 (route: /leisure)
└─ 날씨·레이더 탭   기상청 레이더 + 위성 + 7일 예보             ← 2주차 (route: /weather)
```

> CCTV는 별도 탭이 아닌 **실시간 현황 탭 > 관측소 모달** 안에 임베드 (낙동강홍수통제소 iframe)

### 3.3 수위 판단 기준

**관측소 경보 단계 (낙동강홍수통제소 기준)**

**장소별 위험 판단 기준**

| 장소                   | 안전              | 주의                     | 위험              |
| ---------------------- | ----------------- | ------------------------ | ----------------- |

> 위 기준은 `src/constants/stations.ts`의 `STATIONS`, `PLACES`, `getStationStatus()`, `getPlaceStatus()` 에 구현되어 있음

```
앱 진입

  → 여가 지도 탭 (/leisure) → 내가 가려는 장소 안전 여부 확인
  → 날씨·레이더 탭 (/weather) → 상류 강수 레이더 확인 → 방문 여부 최종 판단
```

| ------------------------- | ----------------------------------------------------------------------------- |

| 기상청 API 오류           | 날씨 카드 "정보 없음" 표시 + 날씨 탭 로딩 스피너                              |

| 레이더 이미지 로드 실패   | SVG 미리보기 fallback 자동 표시                                               |
| AI API 오류               | "AI 안내를 불러오는 중입니다" + 재시도 버튼                                   |

                    ┌──────▼───────────┐     │  CCTV (iframe)       │

                    └──────────────────┘

### 7.2 프론트엔드 (확정 버전)

| 항목            | 버전                    | 비고                                                               |
| --------------- | ----------------------- | ------------------------------------------------------------------ |
| 프레임워크      | React **19**            | `"react": "^19.2.6"`                                               |
| 빌드 도구       | Vite **8**              | `"vite": "^8.0.12"`                                                |
| 언어            | TypeScript **6**        | strict 모드, moduleResolution: bundler                             |
| 라우팅          | React Router **v7**     | `"react-router-dom": "^7.17.0"`, createBrowserRouter               |
| 서버 상태       | TanStack Query **v5**   | `"@tanstack/react-query": "^5.101.0"`, Query Key Factory 패턴      |
| 클라이언트 상태 | Zustand **v5**          | `"zustand": "^5.0.14"`, 순수 UI 상태만                             |
| CSS 프레임워크  | Tailwind CSS **v4**     | `"tailwindcss": "^4.3.1"`, CSS-first 설정, tailwind.config.js 없음 |
| HTTP 클라이언트 | Axios **v1**            | `"axios": "^1.18.0"`, apiClient.ts 중앙화                          |
| 차트            | Recharts **v3**         | `"recharts": "^3.8.1"`                                             |
| 지도            | 카카오맵 JavaScript API | 도메인 제한 설정 필요                                              |
| 폼              | React Hook Form         | 3차 개발 로그인 폼 때 설치 예정                                    |
| 배포            | Vercel                  | GitHub 연동, 자동 배포                                             |

> **TanStack Query 역할 분담**
>
> - `useQuery` — 관측소 수위, 날씨 데이터 fetch + 캐싱
> - `refetchInterval: 600_000` — 10분 자동 폴링
> - `placeholderData` — API 오류 시 마지막 성공 데이터 유지
> - `staleTime` — 동일 데이터 중복 요청 방지
> - Zustand는 지도 마커 선택, 모달 열림 상태 등 순수 UI 상태만 담당

### 8.2 핵심 아키텍처 패턴

**① API 레이어 분리**

```
컴포넌트
  → src/hooks/ (TanStack Query useQuery 호출)
    → src/api/ (Axios 함수, apiClient 사용)
      → src/api/apiClient.ts (Axios 인스턴스, 인터셉터)
```

컴포넌트에서 `apiClient.get()` 직접 호출 금지. 반드시 `hooks/`를 통해서만 데이터 접근.

**② Query Key Factory 패턴**

```ts
// src/queryKeys/index.ts
queryKeys.waterLevels.current(); // ['waterLevels', 'current']
queryKeys.waterLevels.history(id, 24); // ['waterLevels', 'history', id, 24]
queryKeys.weather.current(); // ['weather', 'current']
queryKeys.aiGuide.safety(levels); // ['aiGuide', 'safety', levels]
```

문자열 직접 사용 대신 Factory 함수로 타입 안전하게 키 생성. 오타로 인한 캐시 미스 방지.

**③ Tailwind v4 CSS-first 설정**

```css
/* src/index.css */
@import "tailwindcss";

@theme {
  --color-river: #1d9e75;
  --color-caution: #ef9f27;
  --color-danger: #e24b4a;
  --font-pretendard: "Pretendard", sans-serif;
}
```

`tailwind.config.js` 없음. 커스텀 토큰은 `@theme` 블록 CSS 변수로만 정의.

**④ Zustand 상태 범위**

- `selectedStation` — 관측소 모달 열림/닫힘
- `activeTab` — 현재 활성 탭
- `selectedMapMarker` — 지도 선택 마커 ID
- 서버 데이터(수위, 날씨)는 Zustand에 저장하지 않음 → TanStack Query 캐시가 전담

### 8.3 데이터 흐름

```
백엔드 API (Spring Boot @Scheduled — 10분마다 공공데이터 폴링 후 DB 저장)
  ↓
프론트엔드 TanStack Query (refetchInterval: 600_000 — 10분마다 백엔드 조회)
  ↓
useWaterLevels 훅 → Dashboard / Trends / Leisure 페이지 컴포넌트
  ↓
src/constants/stations.ts의 getStationStatus() / getPlaceStatus() 로 안전 상태 계산
  ↓
UI에서 상태별 컬러·배지·마커 표시
```

Claude API (Anthropic)

**기상청 레이더 이미지**

```
방식     : 기상청 레이더/위성 이미지 URL 직접 embed
종류     : 강수 레이더, 위성 영상, 강수량 분포, 태풍 레이더
fallback : 이미지 로드 실패 시 SVG 미리보기 자동 대체
갱신     : 이미지 URL 10분 주기 새로고침
```
